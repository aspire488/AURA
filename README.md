## Vision
AURA provides a modular, event‑driven cognitive runtime that acts as the intelligence backend for KIO and other client applications. It unifies observation, memory, reasoning, and action into a continuous, observable pipeline.

---

## Core Principles
- **Event‑driven** – every state change is an event.
- **Modular cognition** – independent agents and modules compose capabilities.
- **Persistent memory** – observations and learned artifacts survive restarts.
- **Provider agnostic** – works with OpenAI, OpenRouter, Gemini, Groq, etc.
- **Tool agnostic** – abstracts over n8n, HTTP, database, and custom tools.
- **Recovery‑first** – dead‑letter queue, replay, and snapshot mechanisms.
- **Observable runtime** – rich metrics and health endpoints.

## Major Components
- **Identity** – unique entity tracking for users and agents.
- **Memory** – persisted observations stored in PostgreSQL/Redis.
- **Knowledge** – deterministic facts stored in Neo4j.
- **World Model** – graph representation of entities.
- **Beliefs** – weighted assertions derived from observations.
- **Goals** – desired outcomes generated by agents.
- **Reasoning** – stateless inference over beliefs and goals.
- **Planning** – constructs step‑by‑step action plans.
- **Reflection** – evaluates execution results and generates feedback.
- **Learning** – updates models based on reflection.
- **Strategic Intelligence** – high‑level policy selection (future work).
- **Stewardship** – lifecycle management of resources.
- **Continuity** – ensures state survives restarts.
- **EventBus** – core pub/sub layer with replay and DLQ.
- **Agent Layer** – orchestrator, research, knowledge, planning, coding, browser, automation, communication, monitoring, identity, stewardship.
- **Automation** – n8n integration for external workflow execution.
- **Integrations** – connectors to LLMs, GitHub, Gmail, Calendar, Drive, Discord, Slack, Telegram, WhatsApp, Notion, Dropbox.

---

## Agent Layer
| Agent | Responsibility |
|-------|-----------------|
| **Orchestrator** | Coordinates high‑level tasks and forwards payloads to sub‑agents. |
| **Research** | Retrieves or summarizes external knowledge. |
| **Knowledge** | Stores and retrieves deterministic facts. |
| **Planning** | Generates and updates execution plans. |
| **Coding** | Generates or runs code snippets. |
| **Browser** | Automates web actions via the Chrome extension. |
| **Automation** | Triggers n8n workflows for external side‑effects. |
| **Communication** | Formats messages for chat clients. |
| **Monitoring** | Emits health and metric observations. |
| **Identity** | Manages entity identifiers and authentication data. |
| **Stewardship** | Handles resource cleanup and lifecycle events. |

## Integrations
AURA ships with adapters for the following providers (enabled via configuration in `backend/app/config.py`):
- OpenAI, OpenRouter, Gemini, Groq (LLM back‑ends)
- GitHub (repo events)
- Gmail, Google Calendar, Google Drive
- Discord, Slack, Telegram, WhatsApp
- Notion, Dropbox
All providers are optional and activated only when credentials are supplied.

## Technology Stack
- **Backend**: Python 3.12, FastAPI, SQLAlchemy, asyncpg
- **Databases**: PostgreSQL (relational), Redis (cache), Neo4j (graph), ChromaDB (vector store)
- **Workflow Engine**: n8n (Docker container)
- **Containerisation**: Docker & Docker Compose
- **Frontend**: React 19, Vite, TypeScript, TailwindCSS
- **Browser Extension**: Chrome Manifest V3 (MV3)

## Running AURA
```bash
# Start all services (backend, DBs, n8n, frontend) via Docker Compose
docker compose up -d
```
*Backend* is reachable at `http://localhost:8000`. 
- **Health**: `GET /health` – returns `200 OK` when the service is ready.
- **Metrics**: `GET /metrics` – Prometheus‑compatible metrics.

To run the backend alone (useful for debugging):
```bash
cd backend && uvicorn app.main:app --reload --port 8000
```

---

## KIO Integration
KIO connects to AURA over the WebSocket endpoint `ws://localhost:8000/ws/browser`. It sends user intents and receives observations, keeping cognition external to the client.
